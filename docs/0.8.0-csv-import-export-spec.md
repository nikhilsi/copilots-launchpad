# CSV Import/Export — Design & Technical Spec

**Version:** 0.8.0
**Date:** March 7, 2026
**Status:** Approved for Implementation

---

## Overview

Add the ability to bulk-import accounts from a CSV file and export existing accounts to CSV. Import is the primary use case — the target user has ~20 accounts in a spreadsheet and needs to get them into the app without manual entry.

---

## CSV Format

### Minimum Viable (3 columns)

```csv
label,username,password
Test Premium,alice@contoso.com,P@ss123
Test Basic,bob@contoso.com,P@ss456
```

### Full Format (all columns)

```csv
label,username,password,destination,group,color,notes
Test Premium,alice@contoso.com,P@ss123,Copilot Chat,Admins,#6366F1,Full license
Test Basic,bob@contoso.com,P@ss456,,Testers,,
```

### Column Rules

| Column | Required | Matching | Default if missing |
|--------|----------|----------|--------------------|
| `label` | Yes | — | — |
| `username` | Yes | — | — |
| `password` | Yes | — | — |
| `destination` | No | Matched by label against existing destinations | User picks a default in the import screen |
| `group` | No | — | `"Default"` (editable in import screen) |
| `color` | No | Must be `#RRGGBB` format | `#6366F1` (indigo, editable in import screen) |
| `notes` | No | — | Empty |

- Column order does not matter — matched by header name (case-insensitive)
- Extra columns are ignored

## Import Flow

### Step 1: Trigger

- **Location:** Settings → Accounts tab → "Import CSV" button (next to "Add Account")
- Opens native file picker filtered to `.csv` files
- File is read and parsed in the renderer (no IPC needed for parsing)

### Step 2: Import Preview Screen

A dedicated view replaces the Settings screen during import.

**Layout:**

```
← Back to Settings                              Import Accounts

┌─────────────────────────────────────────────────────────────┐
│  Defaults (applied to all imported accounts)                │
│                                                             │
│  Destination: [ Copilot Chat ▼ ]    Group: [ Default    ]   │
│  Color:  ● ● ● ● ● ● ● ●                                  │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  Preview (12 accounts found, 2 already exist)               │
│                                                             │
│  ☑  Label              Username                   Status    │
│  ─────────────────────────────────────────────────────────  │
│  ☑  Test Premium       alice@contoso.com          Ready     │
│  ☑  Test Basic         bob@contoso.com            Ready     │
│  ☐  Global Admin       admin@contoso.com          Exists    │
│     └─ Already exists — delete first to reimport            │
│  ☑  Helpdesk           help@contoso.com           Ready     │
│  ...                                                        │
└─────────────────────────────────────────────────────────────┘

              [ Cancel ]  [ Import 10 accounts ]
```

### Defaults Bar

- **Destination dropdown** — required. Pre-selected if only one destination exists. If no destinations exist, show a message: "Add a destination first."
- **Group text field** — defaults to `"Default"`, user can type any value
- **Color picker** — 8 preset colors, defaults to indigo (#6366F1)
- These defaults apply to all rows where the CSV did not provide the value

### Preview Table

- Each parsed row shown with: checkbox, label, username, status
- **Ready** — new account, will be imported (checked by default)
- **Exists** — username matches an existing account (unchecked and disabled)
- **Invalid** — missing required field (unchecked and disabled, with reason)
- User can uncheck any "Ready" row to skip it

### Conflict Resolution

Conflicts are detected by matching `username` against existing accounts.

**Design decision:** No overwrite option. Conflicting rows are disabled with the message "Already exists — delete first to reimport." This keeps the user in control and avoids accidental data loss.

### Import Button

- Shows the count of accounts that will actually be imported: "Import N accounts"
- Disabled if count is 0 or no destination is selected
- On click: sends all checked accounts to main process via IPC
- On success: navigates back to Settings → Accounts tab
- Shows success banner: "Imported N accounts"

### Step 3: Execute

- Bulk IPC call to main process with array of accounts
- Each account gets: fields from CSV + defaults for missing fields + generated ID
- Passwords encrypted via safeStorage on the main process side (same as manual add)

## Export Flow

### Trigger

- **Location:** Settings → Accounts tab → "Export CSV" button (next to "Import CSV")
- Opens native save dialog filtered to `.csv`

### Options

- **Include passwords** — checkbox with warning text: "Passwords will be exported in plaintext"
- Default: unchecked (safe export without passwords)

### Output

- All accounts exported with columns: `label, username, password, destination, group, color, notes`
- `destination` column contains the destination label (not ID)
- `password` column is empty unless "Include passwords" is checked
- If passwords are included, they are decrypted from safeStorage before export

---

## IPC Channels

| Channel | Direction | Payload | Purpose |
|---------|-----------|---------|---------|
| `accounts:import` | Renderer → Main | `{ accounts: [{ label, username, password, destinationId, group, color, notes }, ...] }` | Bulk create accounts |
| `accounts:export` | Renderer → Main | `{ includePasswords: bool }` | Returns array of accounts with destination labels |

### Validation (main process)

- `accounts:import` — each account validated with same rules as `accounts:add` (label, username, password required; color format; destination ID must exist)
- Returns `{ imported: N, skipped: N }` summary

## File Parsing

CSV parsing happens in the renderer process (no sensitive data at parse stage — passwords are just strings until they reach the main process for encryption).

**Parser approach:** Simple built-in parser (split by newlines, split by commas, handle quoted fields). No external dependency needed for this scope.

**Edge cases:**
- Commas inside quoted fields: `"Smith, John"` → handled by respecting `"` quoting
- Empty rows → skipped
- Header row required → first row treated as headers
- BOM character (Excel exports) → stripped if present
- Encoding → UTF-8 assumed

## Preload API Additions

```js
// In preload.js
importAccounts: (accounts) => ipcRenderer.invoke('accounts:import', { accounts }),
exportAccounts: (opts) => ipcRenderer.invoke('accounts:export', opts),
```

---

## Security Considerations

- **Import:** Passwords travel from renderer to main via IPC (same as manual account add). Encrypted via safeStorage on arrival.
- **Export with passwords:** Plaintext passwords written to a user-chosen file on disk. Warning shown before export. This is an explicit user action for backup/migration.
- **CSV file:** Not stored by the app — read once during import, then discarded.
- **Validation:** All imported accounts go through the same IPC validation as manually added accounts.

---

## Out of Scope

- Drag-and-drop CSV import (file picker only)
- Importing destinations from CSV (destinations are few and simple to add manually)
- CSV template download (the format is documented above)
- Undo import (user can delete accounts individually)
